# Project Context — Hi, AI! Website

## Overview

This project is a marketing website for **Hi, AI!**, a large Telegram-based media ecosystem focused on AI, technology, and trends.

Hi, AI! operates Telegram AI bots and media channels with a combined audience of 35M+ people.

The website serves as:

- A public company profile
- A credibility and scale showcase
- A gateway for advertisers and partners
- A representation of Hi, AI! as a frontier-tech media company

This is not a SaaS product or dashboard — it is a high-performance media/company website.

---

## Goals

The website must:

- Communicate scale, credibility, and innovation
- Highlight achievements, reach, and ecosystem impact
- Provide clear advertiser-focused sections
- Reflect a modern, frontier-tech brand identity
- Feel like a large media company with a startup spirit

Design inspiration reference:

Morning Brew Inc website structure and tone: https://morningbrewinc.com/

---

## Target Audience

Primary:

- Advertisers
- Media partners
- Business collaborators

Secondary:

- General visitors exploring the brand

---

## Design Direction

The design should feel:

- Minimal and modern
- High-tech and forward-looking
- Clean and editorial
- Confident and premium

Avoid:

- Overly playful UI
- Heavy animations
- Visual clutter

Visual intent:

> Large media credibility + startup agility + tech frontier aesthetic.

---

## Functional Requirements

- Fast loading performance
- SEO-friendly structure
- Clean semantic HTML
- Responsive layout
- Accessibility-aware markup

Current language:

- Cyrillic (primary)

Future:

- Architecture must support easy addition of English localization

---

## Technical Priorities

- Performance first
- Maintainable component structure
- Clean code readability
- Scalable page architecture
- SEO optimization baked into structure

---

## Non-Goals

- No complex web app logic
- No heavy interactivity or dashboard behavior
- No animation-heavy experiences

This is a professional media/company website.

---

## Development Philosophy

When implementing features:

- Prefer simplicity over cleverness
- Optimize for clarity and maintainability
- Avoid overengineering
- Write code that future developers can easily extend

All decisions should support:

→ performance  
→ clarity  
→ scalability  
→ professional presentation
